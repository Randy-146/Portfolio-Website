Pawpal Project Overview

Pawpal is a pet tracker that would keep tabs on your pets health, scheduled activities, and be able to store and go through other pets. The feature was heavy CSS and HTMl, with little coding for JS due to the limitations during the time of coding.

  1. Technology Stack
    • Languages: HTML5, CSS
    • Tools:Github for controlling the commitments and Visual Studio Code for development, Gemini for JS help
    • Deployment: Deployed via netlify for the global web.
  2. Concepts Taught
    • This project expected the use of JS and being able to put together HTML and CSS
    • For JS, we attempted overlays and for animal selection would use JS to animate side-scrolling options.
    • Before this project HTML5 was the main focus, then CSS was taught, and with this project it was expected to put both together to make a structurally sound and good looking website, along with some JS for functionality.
  3. Challenges/Solutions
    • Problem: The settings page was initially broken and wouldn’t navigate for some reason
    • Solution: There was a CSS code that blocked the settings button, removing it allowed the button to work again
    • Problem: The activities was not done correct, and had preset activities
    • Solution: Scrapped the content of the page, and reset and used javascript to set the activities
    • Problem: The scrolling animation worked, but was janky
    • Solution: Replaced the JS animation with CSS animation
  4. Habits of Mind in Action
    • Reflect: The initial project was janky, and had the previous problems listed. Going back to the project, it looks better and functions better than before.
    • Collaborate: Had to collaborate with a partner to bring the page to life, compromised specific designs

EECU Project Overview

EECU was the name of the project, which codes for a budget calculator that would calculate taxes for our Client, EECU. The calculator can take and store the career you choose on the front page, and moving to the next page would give you how much budget you have to spend, and from there you can place any expenses, set or misc. Which will display a pie chart and a tip based on your monthly budgeting.

  1. Technology Stack
    • Languages: HTML5, CSS, and Vanilla Javascript
    • Tools:Github for controlling the commitments and Visual Studio Code for development.
    • Deployment: Deployed via netlify for the global web.
  2. Concepts Taught
    • This project emphasized the use of chart.js, the fetch() API, and a responsive Design
    • fetch() API to fetch data from a set career list
    • chart.js to create a chart based on the user’s choices in budget vs expenses
    • The app is available in a mobile port and will activate when the screen reaches a specific size
  3. Challenges/Solutions
    • Problem:Starting the code off, I messed up the salary, since it wasn’t supposed to be calculated monthly yet, but annually, and so I had to change the bindings
    • Solution:Used ctrl F to find the bindings and changed it from there
    • Problem:Getting the charts to transfer data over was a pain
    • Solution:condensing it made it easier to work on other parts of the project.
    • Problem:Getting the Bar chart to work was a struggle to code
    • Solution:Had to look at previous coding assignments to understand the syntax
  4. Habits of Mind in Action
    • Investigate: Had to skim the code multiple times for Javascript and HTML coding errors, especially when coding the calculations part for the webpage
    • Collaborate: This project was initially coded by other devs and eventually I was able to code it myself, but when others coded my website, I communicated what I wanted.
